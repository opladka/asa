Game Playing

Game playing is one of the oldest and most studied domains in AI. It involves building agents that can make intelligent decisions in competitive environments where outcomes depend on multiple interacting players.

♟

State Space

All possible configurations of the game board or environment at any given point.

🎯

Players

Typically two — MAX (the AI agent trying to win) and MIN (the opponent trying to block).

🏁

Terminal State

The end state where the game concludes — won, lost, or drawn. Utility value is assigned here.

📊

Utility Function

Numeric values assigned to terminal states: +1 win, −1 loss, 0 draw.

Minimax Search Algorithm

Minimax is the foundational algorithm for two-player, zero-sum games. It assumes both players play optimally — MAX tries to maximize the score, MIN tries to minimize it.

1
Generate Game Tree

Expand all states down to terminal (leaf) nodes at the desired depth.

2
Assign Utility

Assign numeric values to all terminal states using the utility function.

3
Back-Propagate

MAX nodes pick the maximum child value; MIN nodes pick the minimum.

4
Choose Best Move

The root node's backed-up value guides the optimal first move for MAX.

function MINIMAX(state, depth, isMaximizing):
    if terminal(state): return utility(state)
    if isMaximizing:
        bestVal = -∞
        for each child of state:
            bestVal = max(bestVal, MINIMAX(child, depth-1, False))
        return bestVal
    else:
        bestVal = +∞
        for each child of state:
            bestVal = min(bestVal, MINIMAX(child, depth-1, True))
        return bestVal
Complexity: Time complexity is O(bm) where b = branching factor and m = depth. For Chess (b≈35, m≈100) this is completely impractical without optimization.

Alpha-Beta Cutoffs

Alpha-Beta pruning is an optimization of Minimax that prunes (skips) branches that cannot possibly affect the final decision, drastically reducing computation without changing the result.

α

Alpha (α)

Best value MAX can guarantee so far. Starts at −∞. Updated at MAX nodes.

β

Beta (β)

Best value MIN can guarantee so far. Starts at +∞. Updated at MIN nodes.

Pruning Rules:
• At a MIN node: if current value ≤ α → prune (MAX already has better)
• At a MAX node: if current value ≥ β → prune (MIN already has better) 
function ALPHA-BETA(state, α, β, isMaximizing):
    if terminal(state): return utility(state)
    if isMaximizing:
        for each child:
            α = max(α, ALPHA-BETA(child, α, β, False))
            if α ≥ β: break  ← BETA cutoff (prune)
        return α
    else:
        for each child:
            β = min(β, ALPHA-BETA(child, α, β, True))
            if β ≤ α: break  ← ALPHA cutoff (prune)
        return β
EnhancementDescriptionBenefit
Move OrderingExamine better moves firstMaximizes pruning opportunities
Iterative DeepeningSearch progressively deeperEfficient time management
Transposition TablesCache evaluated positionsAvoid repeated computation
Best Case Complexity O(bm/2) — effectively doubles the searchable depth vs. plain Minimax

Natural Language Processing

NLP is the branch of AI that enables computers to understand, interpret, and generate human language. It bridges the gap between human communication and machine computation through several layered processing stages.

A
Morphological Analysis

Breaking words into root forms and affixes. "running" → root: "run", suffix: "-ing"

B
Lexical Analysis (Tokenization)

Splitting text into tokens. "AI is powerful." → ["AI", "is", "powerful", "."]

C
Syntactic Analysis (Parsing)

Analyzing grammatical structure using parse trees — mapping sentence constituents (NP, VP, etc.).

D
Semantic Analysis

Understanding meaning. Word Sense Disambiguation: "bank" — river bank vs. financial bank.

E
Pragmatic Analysis

Understanding intent in context. "Can you pass the salt?" is a request, not a yes/no question.

F
Discourse Analysis

Meaning across multiple sentences; resolving co-references ("it", "they", "she").

TechniqueDescription
Bag of WordsRepresent text as word frequency vectors — ignores order
TF-IDFWeight words by importance within and across documents
N-GramsSequence of N words used for language modeling
Word EmbeddingsWord2Vec, GloVe — dense semantic vector representations
TransformersBERT, GPT — attention-based deep learning architecture

Learning, EBL, Discovery & Analogy

Learning allows AI systems to improve performance from experience. Three specialized paradigms — Explanation-Based, Discovery, and Analogy — represent distinct mechanisms for knowledge acquisition.

Explanation-Based Learning (EBL) — A form of deductive learning where the system already has background knowledge and uses a single example to generalize a reusable rule.
1
Observe

Receive a single training example (e.g., "a cup holds water").

2
Explain

Use domain knowledge to explain why the example satisfies the goal.

3
Generalize

Convert the explanation into a general, reusable rule. "Any concave, stable, open-top object holds liquid."

Discovery Learning — The system generates new knowledge by finding patterns from raw data without prior explanations. Examples: BACON (discovered Kepler's Laws), FAHRENHEIT. Includes Empirical Discovery and Conceptual Clustering.
Analogy-Based Learning — Transfer knowledge from a known source domain to a new target domain based on structural/functional similarity. Classic example: Solar system → Atomic structure. Types: Transformational (reuse solution) and Derivational (replay reasoning).

Neural Net Learning

Artificial Neural Networks (ANNs) are inspired by biological neurons and learn by iteratively adjusting connection weights through a process called backpropagation.

Neuron Model Output = f( Σ(wᵢxᵢ) + bias ) where f = activation function (Sigmoid / ReLU / Tanh)
1
Forward Pass

Compute output for a given input batch, layer by layer.

2
Calculate Loss

Compare output to expected values using a loss function (MSE, Cross-Entropy).

3
Backward Pass

Propagate the error back through the network using the chain rule of calculus.

4
Weight Update

Adjust weights using Gradient Descent: w = w − η · ∂L/∂w

Genetic Learning

Genetic Algorithms (GAs) are evolutionary computation techniques inspired by natural selection. They evolve a population of candidate solutions over generations toward an optimal result.

Chromosome

Encoded solution — typically a binary string representing a candidate answer.

Fitness Function

Measures how "good" a solution is. Drives selection pressure in the population.

Crossover

Combine two parent chromosomes at a crossover point to produce offspring.

Mutation

Random bit-flip to maintain genetic diversity and avoid local optima.

Fuzzy Logic Systems

Classical Boolean logic is binary — TRUE or FALSE. Fuzzy Logic deals with degrees of truth, assigning values between 0 and 1 to handle the inherent vagueness in real-world concepts.

Fuzzy Sets

Each element has a membership degree μ ∈ [0, 1]. Shapes: Triangular, Trapezoidal, Gaussian.

Linguistic Variables

Variables with word-based values: "temperature is HOT" instead of "temperature = 95°F".

Fuzzy Rules

IF-THEN rules using linguistic terms: "IF temp IS hot AND humidity IS high THEN fan IS fast"

Defuzzification

Convert the fuzzy output set back to a crisp numeric value using the Centroid method.

1
Fuzzification

Convert crisp input values into fuzzy membership degrees.

2
Rule Evaluation

Apply all fuzzy rules. AND = min operator; OR = max operator.

3
Aggregation

Combine outputs of all rules into a single fuzzy output set.

4
Defuzzification

Convert fuzzy output to a crisp, actionable value (e.g., fan speed = 78%).

Expert Systems

Expert Systems are AI programs that emulate human expert decision-making in a specific domain using a structured knowledge base and an inference engine.

Knowledge Base

Domain-specific facts and IF-THEN production rules representing expert knowledge.

Inference Engine

Applies rules to known facts to derive new conclusions via forward or backward chaining.

Working Memory

Holds current facts, user inputs, and intermediate results during a session.

Explanation Facility

Explains HOW a conclusion was reached and WHY certain questions were asked.

Inference in Bayesian Networks

A Bayesian Network (Belief Network) is a Directed Acyclic Graph (DAG) where nodes represent random variables and directed edges represent probabilistic dependencies between them.

Nodes

Random variables (discrete or continuous) — e.g., Smoking, Cancer, X-Ray Result.

Edges

A→B means A directly influences the probability distribution of B.

CPT

Conditional Probability Table — each node stores P(node | parents).

Joint Probability

P(A,B,C) = P(A) × P(B|A) × P(C|A,B) — product of conditional probabilities.

Inference MethodTypeDescription
Variable EliminationExactEliminate variables by summing out marginals
Junction TreeExactConvert network to tree for efficient computation
MCMC SamplingApproximateGenerate random samples; estimate probabilities
Likelihood WeightingApproximateFix evidence, sample non-evidence variables
Bayes' Theorem P(Cancer | Dyspnoea=yes) = P(Dyspnoea=yes | Cancer) × P(Cancer) ÷ P(Dyspnoea=yes)

K-Means Clustering

K-Means is an unsupervised learning algorithm that partitions n data points into K clusters based on feature similarity by minimizing intra-cluster distance.

1
Initialize

Choose K and randomly place K centroids in the feature space.

2
Assign

Assign each data point to its nearest centroid using Euclidean distance.

3
Update

Recompute each centroid as the mean of all points assigned to its cluster.

4
Converge

Repeat steps 2–3 until centroids stop moving (convergence criterion met).

Objective Function — Minimize J = Σᵢ Σⱼ ‖ x⁽ʲ⁾ᵢ − cⱼ ‖²     d(x,c) = √( Σ(xᵢ − cᵢ)² )
Limitations: Must pre-specify K · Sensitive to initial centroid placement (use K-Means++ for better initialization) · Poor with non-spherical or uneven-density clusters · Sensitive to outliers.

Machine Learning

Machine Learning is a subset of AI that enables systems to learn patterns from data and make decisions with minimal human intervention across three fundamental paradigms.

📚

Supervised Learning

Learns from labeled data. Learn mapping f: X→Y. Tasks: Classification, Regression. Algorithms: SVM, Decision Trees, Neural Networks.

🔍

Unsupervised Learning

Learns from unlabeled data. Finds hidden structure. Tasks: Clustering, Dimensionality Reduction. Algorithms: K-Means, PCA, DBSCAN.

🎮

Reinforcement Learning

Agent learns by interacting with environment, receiving rewards. Goal: maximize cumulative reward. Algorithms: Q-Learning, PPO, DQN.

Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') - Q(s,a)]
α = learning rate   γ = discount factor
r = immediate reward   s' = next state
ConceptDescription
OverfittingModel memorizes training data; poor generalization. Fix: dropout, regularization.
UnderfittingModel too simple; high bias, low variance. Fix: increase capacity.
Cross-ValidationRotating train/test splits for more reliable evaluation estimates.
Bias-Variance TradeoffBalancing model complexity against generalization ability.
Evaluation Metrics Precision = TP/(TP+FP)  ·  Recall = TP/(TP+FN)  ·  F1 = 2·(P·R)/(P+R)
